# Agent Start Here

Use this repo when the task is about:

- monitoring, logging, metrics, and observability documentation in `dirac-bia`
- dashboard and IT-systems documentation scoped to this repo
- determining which repo-local docs matter before broader DIRAC framework guidance

Read in this order:

1. `docs/context/capabilities.md`
2. `docs/context/workflows.md`
3. `docs/context/contracts.md`
4. `docs/adr/index.md`
5. `README.md`

Use this repo to answer:

- What monitoring/observability topics are documented here?
- Which pages should be updated when dashboard or monitoring guidance changes?
- Which repo-local ADR proposal documents the current Dirac-BIA stack direction?
- What repo-local contracts matter before broader framework guidance?

Do not use this repo as the source of truth for cross-DIRAC architecture
decisions that belong in `dirac-framework`.
